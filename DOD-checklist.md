# Definition of Done (DoD) Checklist

## 功能完成度

- [ ] 所有 P0 功能已实现并测试通过
- [ ] 所有 P1 功能已实现并测试通过
- [ ] 所有 P2 功能已实现并测试通过
- [ ] 无阻塞性缺陷 (P0/P1 级别)
- [ ] 所有已知的 P2/P3 缺陷已记录

## 代码质量

- [ ] 代码风格符合 ESLint 规则
- [ ] TypeScript 类型标注完整，无 `any` 类型滥用
- [ ] Rust 代码遵循最佳实践
- [ ] 所有代码经过代码审查
- [ ] 无安全漏洞 (OWASP Top 10)

## 测试覆盖

- [ ] 单元测试覆盖率 >= 70%
- [ ] 集成测试覆盖核心业务流程
- [ ] E2E 测试覆盖主要用户场景
- [ ] 所有测试通过 (CI 绿灯)
- [ ] 性能测试达标

## 文档完整

- [ ] API 文档完整并更新
- [ ] 组件使用文档完整
- [ ] 部署文档完整
- [ ] Release Notes 编写完成
- [ ] CHANGELOG 更新

## 发布准备

- [ ] 版本号符合语义化版本 (SemVer)
- [ ] 变更日志已更新
- [ ] 安装包已生成并验证
- [ ] 回滚方案已准备
- [ ] 监控告警已配置

## 合规性

- [ ] 许可证合规
- [ ] 第三方依赖许可证检查通过
- [ ] 安全策略配置正确
- [ ] CSP 配置生效

- [ ] QA 负责人确认: _______________
- [ ] 产品负责人确认: _______________
